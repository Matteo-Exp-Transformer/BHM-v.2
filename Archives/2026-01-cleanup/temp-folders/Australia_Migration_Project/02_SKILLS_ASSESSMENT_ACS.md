## ‚úçÔ∏è COME SCRIVERE IL RPL PROJECT REPORT

### Struttura di ogni progetto:

**1. Project Overview**
- Nome progetto
- Periodo (date precise)
- Azienda/Cliente
- Il tuo ruolo specifico

**2. Technical Details**
- Linguaggi di programmazione usati
- Database e tecnologie
- Metodologie (Agile, Waterfall, ecc.)
- Strumenti e framework

**3. Your Contribution**
- Cosa hai fatto TU (non il team)
- Problemi risolti
- Decisioni tecniche prese
- Risultati misurabili

**4. Key Areas Demonstrated**
- Collegare a CBOK areas
- Es: "Database design" ‚Üí descrivere schema, normalizzazione
- Es: "Quality assurance" ‚Üí descrivere testing, code review

### Tips importanti:
- **NO copia-incolla** - ACS usa Turnitin
- **NO lavori "finti"** - verificheranno
- Sii specifico e tecnico
- Usa terminologia professionale
- Quantifica dove possibile (es. "ottimizzato query del 40%")
